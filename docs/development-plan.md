# Markdown Viewer 개발 및 테스트 계획

## 문서 목적

- `skimd`의 초기 구현 범위(MVP)와 단계별 개발 순서를 정리한다.
- 터미널/SSH 환경에서 쓸 수 있는 TUI 마크다운 뷰어를 목표로 한다.
- `~/ecode/mux`의 구조와 `~/.tmux.conf`의 popup 진입 방식을 참고 기준으로 사용한다.

## 참고 기준

- `~/ecode/mux/main.go`
- `~/ecode/mux/internal/ui/app.go`
- `~/ecode/mux/internal/ui/layout.go`
- `~/ecode/mux/internal/ui/layout_test.go`
- `~/.tmux.conf`

확인된 popup 패턴:

```tmux
bind m display-popup -E -w 92% -h 88% "/home/euteum-park/go/bin/mux"
```

이 프로젝트도 같은 방식으로 `display-popup` 진입점을 제공하는 방향이 적합하다.

## 목표와 비목표

### 목표

- 디렉토리 리스트를 탐색하고 상/하위 디렉토리로 이동할 수 있다.
- 현재 디렉토리에서 마크다운 파일을 선택해 렌더링된 형태로 볼 수 있다.
- 긴 문서는 스크롤해서 읽을 수 있다.
- 일반 터미널과 tmux popup 환경에서 모두 동작한다.

### 비목표

- 마크다운 편집 기능
- 실시간 파일 변경 감지
- 브라우저 수준의 완전한 HTML/CSS 렌더링
- 이미지 인라인 렌더링

## 기술 방향

- 언어: Go
- TUI 프레임워크: Bubble Tea
- UI 컴포넌트: Bubbles `viewport`, `textinput`(필요 시)
- 스타일링: Lip Gloss
- 마크다운 렌더링 후보: `glamour`
- 폭 계산/레이아웃 안정화: `github.com/charmbracelet/x/ansi` 계열 유틸 재사용 검토

선정 이유:

- `mux`가 이미 같은 기술 스택으로 구성되어 있어 구조를 재사용하기 쉽다.
- SSH/tmux 환경에서 의존성이 가볍고 배포가 단순하다.
- 렌더링과 UI 상태 관리를 분리하기 좋다.

## 화면/상태 설계

### 기본 화면

- 좌측 35~40%: 디렉토리 엔트리 리스트
- 우측 60~65%: 선택한 마크다운의 렌더링 결과
- 하단: 키바인딩 도움말
- 상단: 현재 경로, 선택 파일명, 에러/상태 메시지

### 엔트리 규칙

- 디렉토리 우선 정렬
- 그 다음 마크다운 파일 정렬
- 나머지 파일은 하단에 표시하거나, MVP에서는 흐리게 표시
- 숨김 파일은 MVP에서는 그대로 표시하되, 필요 시 이후 토글 추가

### 모드 제안

- `browse`
  - 좌측 리스트 탐색
- `preview`
  - 우측 문서 스크롤
- `error`
  - 읽기 실패, 권한 오류, 렌더 오류 표시

초기 버전에서는 모드를 최소화하는 편이 낫다. 추천 흐름은 다음과 같다.

- 방향키 또는 `j/k`로 좌측 엔트리 이동
- `Enter`로 디렉토리 진입 또는 마크다운 열기
- `Backspace`, `h`, `Left`로 상위 디렉토리 이동
- `Tab`으로 브라우저/프리뷰 포커스 전환
- 프리뷰 포커스에서 `j/k`, `PgUp/PgDn`, `g/G`로 스크롤
- `r`로 새로고침
- `q`로 종료

## 단계별 개발 계획

### 1단계: 프로젝트 부트스트랩

- Go 모듈 초기화
- `mux`와 유사한 기본 Bubble Tea 앱 골격 구성
- 좌/우 2패널 고정 레이아웃과 help bar만 먼저 구현

완료 기준:

- 빈 화면이 아니라 현재 경로와 패널 골격이 뜬다.
- 터미널 리사이즈 시 레이아웃이 깨지지 않는다.

### 2단계: 파일 브라우저 구현

- 시작 경로를 기준으로 `os.ReadDir` 기반 탐색 구현
- 디렉토리/파일 타입 구분
- 마크다운 확장자 판별: `.md`, `.markdown` 우선 지원
- 커서 이동, 디렉토리 진입, 상위 디렉토리 복귀 구현

완료 기준:

- 키보드만으로 디렉토리를 계속 이동할 수 있다.
- 현재 디렉토리의 마크다운 파일을 선택할 수 있다.

### 3단계: 마크다운 렌더링 구현

- 선택한 파일을 읽어 우측 패널에 렌더링
- 우측 패널 폭 기준으로 다시 렌더링
- 긴 문서는 viewport 스크롤 지원
- 렌더 실패 시 원문 텍스트 fallback 또는 에러 메시지 표시

완료 기준:

- 제목, 리스트, 코드 블록, 인용문, 표가 읽기 가능한 수준으로 출력된다.
- 80x24 같은 좁은 터미널에서도 깨지지 않는다.

### 4단계: 사용성 보강

- 상단 breadcrumb 또는 현재 경로 표시 개선
- 현재 선택 항목의 타입/크기/수정 시간 표시
- 렌더 캐시 추가
  - 키: `path + width + file mtime`
- 비마크다운 파일 선택 시 안내 메시지 표시

완료 기준:

- 같은 파일을 다시 열 때 불필요한 재렌더가 줄어든다.
- 에러/상태 메시지가 사용자에게 충분히 보인다.

### 5단계: tmux popup 진입점 추가

- 시작 디렉토리 인자 지원
  - 예: `skimd .`
  - 예: `skimd /path/to/docs`
- tmux popup용 실행 예시 제공

예상 바인딩:

```tmux
bind v display-popup -E -w 92% -h 88% -d "#{pane_current_path}" "/home/euteum-park/go/bin/skimd ."
```

또는 현재 패스만 넘기고 앱 내부에서 절대 경로로 정규화한다.

완료 기준:

- tmux 안에서 현재 pane 경로 기준으로 popup이 열린다.
- popup 종료 후 원래 세션/패널 흐름이 깨지지 않는다.

### 6단계: 문서화와 패키징

- README 작성
- 키바인딩/설치 방법 정리
- tmux binding 예시 제공
- 샘플 스크린샷 또는 asciinema 캡처 준비

## 주요 설계 결정

### 1. `mux`처럼 UI와 외부 연동을 분리한다

tmux popup은 진입 방식일 뿐 핵심 제품 기능이 아니다. popup 연동은 별도 패키지 또는 별도 실행 예시로 두고, 코어 앱은 일반 터미널에서도 동일하게 동작해야 한다.

### 2. 렌더링은 파일 선택 시점에만 수행한다

커서를 옮길 때마다 모든 파일을 렌더링하면 큰 문서에서 성능이 흔들릴 수 있다. 초기 버전은 `Enter`로 연 파일만 렌더링하고, 필요하면 나중에 hover preview를 추가한다.

### 3. 레이아웃 테스트를 초기에 넣는다

`mux/internal/ui/layout_test.go`처럼 다양한 터미널 폭/높이에서 줄 수와 패널 높이가 깨지지 않는지를 초기에 보장해야 한다. TUI는 기능보다 레이아웃 회귀가 더 자주 발생한다.

## 테스트 계획

### 자동화 테스트

#### 단위 테스트

- `internal/browser`
  - 디렉토리/파일/마크다운 분류가 맞는지
  - 정렬 규칙이 디렉토리 우선으로 동작하는지
  - 상위 디렉토리 계산이 루트/홈 경계에서 안전한지
- `internal/render`
  - 폭이 바뀌면 다시 렌더링되는지
  - 코드 블록/표/한글/긴 줄에서 panic 없이 동작하는지
  - 존재하지 않는 파일, 권한 오류, 빈 파일 처리
- `internal/ui`
  - cursor bound 검사
  - focus 전환 검사
  - 레이아웃 결과가 터미널 높이를 넘지 않는지

#### 골든 테스트

- `testdata/docs/simple.md`
- `testdata/docs/table.md`
- `testdata/docs/codeblock.md`
- `testdata/docs/unicode.md`

검증 방식:

- ANSI escape를 제거한 정규화 결과를 golden 파일과 비교
- 폭 80, 100, 120 기준으로 스냅샷을 분리

주의점:

- ANSI 전체 문자열 비교는 테마 변경에 취약하므로, 텍스트 구조와 줄바꿈 안정성을 더 중요하게 본다.

#### 통합 테스트

- 임시 디렉토리 트리를 만든 뒤 탐색 시나리오 실행
- 하위 디렉토리 진입 후 마크다운 파일 열기
- viewport 스크롤 후 상태 유지 확인
- 파일 수정 시간 변경 시 캐시 무효화 확인

### 수동 테스트

#### 기본 탐색

- 3단계 이상 중첩된 디렉토리를 문제없이 이동하는지
- 디렉토리와 마크다운 파일이 섞여 있어도 선택이 직관적인지
- 빈 디렉토리에서 정상 메시지를 보여주는지

#### 문서 렌더링

- 제목, 리스트, 체크박스, 코드 블록, 표, 인용문
- 긴 문서 스크롤
- 매우 긴 한 줄 텍스트
- 한글/영문 혼합 문서

#### 예외 상황

- 읽기 권한이 없는 파일
- 0바이트 파일
- 1MB 이상 큰 문서
- 확장자는 `.md`지만 내용이 깨진 파일
- 심볼릭 링크

#### 환경별 검증

- 일반 로컬 터미널
- SSH 접속 터미널
- tmux 내부 일반 창
- tmux popup
- 80x24, 120x40, 160x50 해상도

### tmux popup 전용 확인 항목

- popup이 현재 pane 작업 디렉토리에서 시작하는지
- popup 내부에서도 alt screen이 정상 동작하는지
- 종료 시 prompt/화면 복원이 자연스러운지
- tmux가 없는 환경에서도 앱 자체는 정상 실행되는지

## 완료 기준

- 사용자가 터미널에서 디렉토리를 탐색해 원하는 마크다운 파일을 열 수 있다.
- 열람 중인 문서를 스크롤해서 읽을 수 있다.
- 레이아웃이 여러 터미널 크기에서 안정적으로 유지된다.
- tmux popup으로 바로 진입할 수 있는 실행 예시가 준비된다.
- 기본 단위 테스트와 핵심 통합 테스트가 CI 없이도 로컬에서 재현 가능하다.

## 권장 구현 순서 요약

1. `mux` 구조를 따라 2패널 TUI 골격을 만든다.

3. 마크다운 렌더러와 viewport 스크롤을 붙인다.
4. 레이아웃/렌더 테스트를 먼저 고정한다.
5. 마지막에 tmux popup 진입점을 붙인다.
