# 프로젝트 이름 후보

이 문서는 최종 이름을 정하기 전에 검토한 후보 기록입니다.
최종 선택은 `skimd`입니다.

이 프로젝트의 이름 기준은 아래 다섯 가지입니다.

- 터미널에서 바로 치기 쉬워야 한다
- `mux`와 같이 놓였을 때 어색하지 않아야 한다
- “마크다운을 편집”보다 “빠르게 읽고 검토”하는 느낌이 살아야 한다
- 너무 범용적이지 않아 검색성과 패키지명이 너무 약해지지 않아야 한다
- tmux popup 도구라는 흐름과 잘 맞아야 한다

## 추천 후보

### 1. `skimd` (추천)

`skim` + `md`

장점:

- “문서를 빠르게 훑는다”는 용도가 바로 드러난다
- `md`가 붙어서 markdown 도구라는 점이 명확하다
- CLI 이름으로 짧고 기억하기 쉽다
- `mux`와 같이 두었을 때 결이 잘 맞는다

주의:

- 조금 기능 중심 이름이라 브랜드성은 아주 강하지 않다

추천 사용:

- repo: `skimd`
- binary: `skimd`

### 2. `folio`

문서 묶음, 페이지 모음이라는 느낌이 있는 이름입니다.

장점:

- 가장 브랜드형 이름에 가깝다
- 문서 도구라는 인상이 자연스럽다
- 향후 markdown viewer 이상으로 확장해도 버티기 쉽다

주의:

- popup review 도구라는 성격은 이름만 보고 바로 드러나지 않는다

추천 사용:

- repo: `folio`
- binary: `folio`

### 3. `glance`

빠르게 확인한다는 사용 흐름과 잘 맞는 이름입니다.

장점:

- tmux popup에서 잠깐 열어 보는 컨셉과 매우 잘 맞는다
- “AI가 만든 문서를 빠르게 리뷰”하는 인상이 강하다

주의:

- CLI 이름으로는 조금 길다
- 검색성과 패키지 충돌 가능성이 높다

### 4. `sift`

문서를 걸러 보고 점검한다는 느낌의 이름입니다.

장점:

- AI가 많이 만든 문서들 사이를 오가며 검토하는 워크플로우와 맞는다
- 짧고 CLI로 쓰기 편하다

주의:

- markdown viewer라는 점이 직접적으로 드러나지 않는다

### 5. `leaf`

페이지를 넘겨 읽는 느낌이 있는 짧은 이름입니다.

장점:

- 매우 짧고 기억하기 쉽다
- 문서를 읽는 도구라는 인상은 있다

주의:

- 너무 일반적인 이름이라 검색성과 충돌 면에서 약하다

## 결론

현재 기준으로는 `skimd`가 가장 균형이 좋습니다.

이유:

- popup review 도구라는 핵심 용도를 잘 설명한다
- markdown 도구라는 점이 이름에 남는다
- `mux`와 함께 써도 톤이 맞는다
- CLI 이름으로도 충분히 짧다

브랜드성까지 더 보고 싶다면 차선은 `folio`입니다.

## 추천 정리

- 실용 우선: `skimd`
- 브랜드 우선: `folio`
- 현재 작업 이름 유지: `skimd`는 개발 중 임시 바이너리명으로만 사용
